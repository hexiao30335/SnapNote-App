# SnapNote 云端打包 APK 说明

## 方案：GitHub Actions 自动构建

已配置好 GitHub Actions 工作流，推送到 GitHub 仓库后即可自动构建 APK。

---

## 操作步骤

### 1. 注册 GitHub 账号
访问 https://github.com 注册（免费）。

### 2. 创建仓库
- 点击右上角 `+` → `New repository`
- 仓库名填 `SnapNote`
- 选择 `Public`（公开仓库 Actions 免费）
- 点击 `Create repository`

### 3. 上传代码
在仓库页面，点击 `uploading an existing file`，将 `SnapNote` 文件夹内的所有文件拖拽上传，然后点击 `Commit changes`。

或者使用 Git 命令行：
```bash
cd SnapNote
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/SnapNote.git
git push -u origin main
```

### 4. 触发构建
上传完成后，GitHub Actions 会自动开始构建。你也可以：
- 进入仓库 → `Actions` 标签
- 点击 `Build APK` 工作流
- 点击右侧 `Run workflow` → `Run workflow` 手动触发

### 5. 下载 APK
构建完成后：
- 进入仓库 → `Actions` → 最新的构建记录
- 页面底部 `Artifacts` 区域
- 点击 `snapnote-debug-apk` 下载 ZIP
- 解压后即可得到 `app-debug.apk`

或者查看 `Releases` 页面，每个版本也会附带 APK 下载。

## 构建配置说明

| 配置项 | 说明 |
|--------|------|
| 触发条件 | push 到 main/master 分支、pull_request、手动触发 |
| JDK 版本 | 17 (Temurin) |
| Gradle 版本 | 8.4 |
| 构建命令 | `./gradlew assembleDebug` |
| 产物 | `app-debug.apk` |
| 缓存 | Gradle 依赖缓存加速构建 |

---

## 常见问题

**Q: 构建失败提示 `gradlew` 没有执行权限？**
A: 工作流中已配置 `chmod +x gradlew`，通常不需要手动处理。

**Q: 如何构建 Release APK？**
A: 修改 `.github/workflows/build-apk.yml` 中的 `./gradlew assembleDebug` 为 `./gradlew assembleRelease`，但需要配置签名密钥。

**Q: 构建需要多长时间？**
A: 首次构建约 5-10 分钟（下载依赖），后续构建约 2-3 分钟（有缓存）。
